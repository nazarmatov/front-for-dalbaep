## 🏗️ Folder Organization & Conventions

### **src/components/**
- **Purpose:** House all reusable React components
- **Structure:** Each component gets its own folder with `.jsx` and `.css` files
- **Naming Convention:**
  - Folder name: PascalCase (e.g., `Button`, `Header`, `Navigation`)
  - File names: 
    - `ComponentName.jsx` — React component
    - `ComponentName.css` — Component-specific styles
  
**Example Structure:**
```
src/components/
├── App/
│   ├── App.jsx
│   └── App.css
├── Button/
│   ├── Button.jsx
│   └── Button.css
├── Header/
│   ├── Header.jsx
│   └── Header.css
└── Footer/
    ├── Footer.jsx
    └── Footer.css
```

### **src/styles/**
- **Purpose:** Global styles, CSS variables, and resets
- **Files:**
  - `index.css` — Main global stylesheet (imported in `src/main.jsx`)
  - All other files: utility/theme styles as needed

### **src/assets/**
- **Purpose:** Static files (images, SVGs, icons, fonts, etc.)
- **Naming Convention:** kebab-case or lowercase (e.g., `hero-image.png`, `icon-menu.svg`)

### **src/pages/** (Future)
- **Purpose:** Page-level components (full-page layouts)
- **Naming Convention:** PascalCase folders, one `.jsx` + one `.css` per page
- **Example:**
  ```
  src/pages/
  ├── HomePage/
  │   ├── HomePage.jsx
  │   └── HomePage.css
  └── ContactPage/
      ├── ContactPage.jsx
      └── ContactPage.css
  ```

## 📝 File Format Examples

### **React Component (.jsx)**

```jsx
// src/components/Button/Button.jsx
import './Button.css'

function Button({ label, onClick }) {
  return (
    <button className="btn" onClick={onClick}>
      {label}
    </button>
  )
}

export default Button
```

### **Component Styles (.css)**

```css
/* src/components/Button/Button.css */
.btn {
  padding: 10px 20px;
  background-color: #007bff;
  color: white;
  border: none;
  border-radius: 4px;
  cursor: pointer;
  font-size: 14px;
  transition: background-color 0.3s ease;
}

.btn:hover {
  background-color: #0056b3;
}
```

### **Global Styles (.css)**

```css
/* src/styles/index.css */
:root {
  --primary-color: #007bff;
  --text-color: #333;
  --bg-color: #fff;
}

* {
  box-sizing: border-box;
}

body {
  font-family: system-ui, -apple-system, sans-serif;
  color: var(--text-color);
  background-color: var(--bg-color);
}
```

## 💡 Import Conventions

### **Importing Components**
```jsx
// Always use relative paths
import Button from '../Button/Button.jsx'
import Header from '../Header/Header.jsx'

// Or from parent folders
import App from '../../components/App/App.jsx'
```

### **Importing Styles**
```jsx
// Inside a component file
import './ComponentName.css'

// In main entry (global styles)
import './styles/index.css'
```

### **Importing Assets**
```jsx
import imagePath from '../assets/image.png'
// Then use: <img src={imagePath} alt="description" />
```

## ✍️ Writing Code for This Project

### **Do's** ✅
- Use functional components with hooks
- Create one component per folder
- Always co-locate styles with components (same folder)
- Use PascalCase for component names and folders
- Import styles at the top of component files
- Keep components small and reusable
- Use CSS classes for styling (not inline styles)
- Export default from component files

### **Don'ts** ❌
- Don't mix multiple components in one file
- Don't use older class components
- Don't create new folders without purpose
- Don't forget to import `.css` files
- Don't use inconsistent naming conventions
- Don't commit `node_modules/` or `dist/`

## 🎯 How to Extend the Project

### **Adding a New Component**
1. Create folder: `src/components/MyComponent/`
2. Create files:
   - `MyComponent.jsx` — React component
   - `MyComponent.css` — Styles
3. Export the component as default
4. Import and use in other components

### **Adding a New Page**
1. Create folder: `src/pages/MyPage/`
2. Create files:
   - `MyPage.jsx`
   - `MyPage.css`
3. Use in router or in App component

### **Adding Global CSS Variables**
- Edit `src/styles/index.css`
- Add to `:root` section
- Reference with `var(--variable-name)` in any component CSS

## 🔍 Quick Reference for AI

When generating code for this project:
1. **Always check** if a file should live in `src/components/` or `src/pages/`
2. **Always pair** `.jsx` files with `.css` files in the same folder
3. **Always import** styles at the top of component files
4. **Always use** relative imports (e.g., `'./MyComponent.css'`)
5. **Always export default** from component files
6. **Never create** inline styles—use CSS classes
7. **Never ignore** ESLint warnings—follow the configured rules
